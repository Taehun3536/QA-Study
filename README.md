# [QA Test-Bed] 게시판 CRUD 및 비즈니스 로직 검증

[cite_start]1년 5개월간 물류 플랫폼 QA로서 812건 이상의 결함을 관리하며 쌓은 실무 감각을 바탕으로, 수동 테스트의 한계를 기술적으로 해결하기 위해 진행 중인 **테스트 자동화 학습 프로젝트**입니다. [cite: 12, 39, 49, 61, 67]

단순한 게시판 기능을 넘어, 실제 서비스의 핵심인 **CRUD(생성·조회·수정·삭제) 라이프사이클**에서의 데이터 무결성을 확보하고 예외 상황을 제어하는 데 목적을 두고 있습니다.

## 🛠 Tech Stack
* [cite_start]**Language:** Java [cite: 25]
* [cite_start]**Framework:** Spring Boot [cite: 13]
* [cite_start]**Testing:** JUnit5, MockMvc, Playwright [cite: 14, 26, 49, 62]
* [cite_start]**Tools:** GitHub, Jira, Notion [cite: 14]

## 🎯 주요 테스트 관점 및 학습 내용

### 1. 비즈니스 로직 무결성 검증 (JUnit5, MockMvc)
* **CRUD 데이터 정합성:** 게시글이 생성되고 수정, 삭제되는 과정에서 데이터베이스와 서버 간의 데이터가 누락 없이 일치하는지 확인합니다.
* **API 레벨 테스트:** MockMvc를 활용하여 각 API의 HTTP 상태 코드와 응답 데이터가 기획서의 의도대로 반환되는지 검증합니다.

### 2. 사용자 관점의 예외 상황(Edge Case) 대응
* **경계값 분석:** 제목이나 본문의 글자 수 제한, 특수문자 입력 등 데이터 입력 시 발생할 수 있는 한계 상황을 테스트 시나리오로 구성하여 코드 레벨에서 방어합니다.
* **비정상 흐름 제어:** 로그인하지 않은 사용자의 접근이나 권한이 없는 글의 수정/삭제 시도 등 보안 및 권한 로직의 빈틈을 체크합니다.

### 3. 수동 QA 경험의 자동화 치환 (Playwright)
* [cite_start]**회귀 테스트(Regression Test) 효율화:** 실무에서 매번 수동으로 반복했던 핵심 기능(Smoke Test)을 Playwright 스크립트로 작성하여 테스트 생산성을 높이는 연습을 하고 있습니다. [cite: 52]
* [cite_start]**E2E 시나리오 검증:** 사용자가 게시판을 이용하는 전체 여정(글 작성 -> 목록 확인 -> 수정 -> 삭제)이 크로스 브라우징 환경에서 정상 작동하는지 확인합니다. [cite: 58]

## 📈 성장 목표
* [cite_start]현재의 게시판 구조를 바탕으로 **CI/CD 파이프라인(GitHub Actions)**과 연동된 자동화 환경을 구축할 예정입니다. [cite: 52]
* [cite_start]단순 기능 검증을 넘어, 실무에서 경험했던 **가상계좌 결제나 플랫폼 간 동기화**와 같은 복잡한 도메인 로직을 자동화 코드로 완벽히 재현하는 것을 목표로 합니다. [cite: 54, 57, 65]
