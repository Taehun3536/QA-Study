## 프로젝트 개요

Spring Boot 기반 게시판 서비스에 대해  
**API 통합 테스트와 UI E2E 테스트를 분리 설계**하여  
사용자 인증 상태, 권한 제어, 데이터 상태 전이, 그리고 주요 예외 상황을 검증한  
QA 자동화 테스트 프로젝트입니다.

- API 테스트에서는 MockMvc를 활용해  
  로그인 여부, 작성자 권한, 데이터 생성/수정/삭제에 따른  
  **HTTP 상태 코드와 응답 데이터의 정합성**을 검증했습니다.

- UI 테스트에서는 Playwright를 사용해  
  실제 사용자 흐름(회원가입 → 로그인 → 게시글 작성/조회/수정/삭제)을 기반으로  
  **페이지 전환, 세션 유지/만료, 비로그인 접근 제한** 등  
  상태 전이가 화면에 올바르게 반영되는지를 검증했습니다.

이를 통해 단순 기능 동작 확인이 아닌,  
“사용자 상태 변화에 따라 시스템이 올바르게 반응하는지”를  
API와 UI 양쪽 관점에서 입체적으로 검증하는 것을 목표로 했습니다.


## 테스트 전략

본 프로젝트는 테스트 피라미드 관점에서
API 통합 테스트와 UI E2E 테스트를 분리하여 설계했습니다.

- API 통합 테스트 (MockMvc)
    - 비즈니스 로직, 권한, 데이터 무결성 검증
    - MockMvc + 세션 기반 테스트 (Controller ~ DB 흐름)

- UI E2E 테스트 (Playwright)
    - 실제 사용자 플로우 검증
    - Playwright 기반 브라우저 자동화


## API 통합 테스트

### 사용 기술
- Spring Boot
- MockMvc
- JUnit5
- H2 / Test Profile

### 테스트 범위
- 로그인 여부에 따른 게시글 생성
- 작성자/비작성자 권한 검증 (403)
- 게시글 수정/삭제 후 상태 검증
- 입력값 검증 (400)
- 삭제 후 재조회 시 404

### 특징
- 테스트 간 독립성을 위해 사용자/게시글 생성 헬퍼 사용
- 세션 기반 인증 흐름 검증
- Given-When-Then 구조로 가독성 유지


## 테스트 실행 방법

./gradlew test

### UI(E2E) 테스트
./gradlew e2eTest
